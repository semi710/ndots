## Before You Code

### Search first
Do not repeat yourself. Before creating anything new, check if it exists - utilities, functions, patterns. If it seems like it should exist, it probably does. Prefer importing over copying. Extract shared logic into reusable abstractions only when the duplication is real.

### Respect existing standards
Scan the codebase for conventions before writing. Look for README, CONTRIBUTING, .github/, docs/. Match naming, indentation, error handling, testing patterns. Do not assume, do not skip this for "simple" tasks, do not introduce new patterns unless asked.

### Think before acting
For complex problems, do not jump to implementation. Restate the problem, identify constraints, explore 2-3 approaches with tradeoffs, ask if ambiguous, recommend with justification, then implement if approved. Prefer simplicity over cleverness, explicit over implicit, small changes with clear impact. Avoid premature optimization, over-engineering, changing unrelated code, assuming context you don't have.
