## Respect Existing Standards

### Mandatory Before Starting Any Task
Before writing code, documentation, tests, or making changes:

1. **Scan the codebase** for existing conventions and patterns
2. **Look for standard documents** in the project root:
   - `README.md`, `CONTRIBUTING.md`, `CONVENTIONS.md`
   - `.github/`, `docs/`, or similar directories
   - Any files mentioning code style, standards, or guidelines
3. **Match existing style** - observe how the project currently does things:
   - Naming conventions (camelCase vs snake_case)
   - Indentation and formatting patterns
   - Error handling approaches
   - Testing patterns
4. **Apply those standards** to all work produced
5. **Verify compliance** before declaring completion

### Do NOT Assume
- Do not assume you know the project's conventions
- Do not skip context scanning for "simple" tasks
- Do not produce work that doesn't match established patterns
- Do not introduce new patterns unless explicitly requested
